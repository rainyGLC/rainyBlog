# Git宝典

# 账户配置

  1. 配置全局账户，对所以Git仓库有效
git config --global user.name 'Your Name'
git config --global user.email 'Your Email'
1
2
  1. 配置局部账户，只对当前Git仓库有效
git config --local user.name 'Your Name'
git config --local user.email 'Your Email'
1
2
  1. 查看全局配置
git config --global --list
1
  1. 查看局部配置
git config --local --list
1

# 本地操作

# 初始化本地库

git init
1

# 查看状态

git status
1

# 添加暂存区

  • 将当前目录及其子目录下所有变更添加到暂存区
git add .
1
  • 将本地库所有变更添加到暂存区
git add -A
1
  • 指定文件添加暂存区
git add file1 file2 ...
1

# 提交所有变更

  • 提交所有变更
git commit
# or
git commit -m '日志信息' .
1
2
3
  • 提交指定文件
git commit -m '日志信息' filename
1
2

# 比较差异 git diff

  1. 比较工作区和暂存区的所有差异,只能查看旧文件的变更(包括修改和删除)，不能查看新文件(因为新文件还未被git 追踪)
git diff
1
  1. 比较指定文件工作区和暂存区的差异
git diff fileName
1
  1. 比较暂存区和HEAD的所有差异
git diff --cached
1
  1. 比较指定文件暂存区和HEAD的差异
git diff --cached fileName
1
  1. 比较两个版本的差异
  • 以前者未基准看后者的变化
  • HEAD 表示最后一次 commit 对应的版本，HEAD~1 往前一个版本
git diff 版本号1 版本号2
1
git diff HEAD~1 HEAD
git diff HEAD~2 HEAD
1
2
  1. 比较两个分支指定文件的差异
git diff 分支1 分支2 fileName
1

# 查看日志信息 git log

  1. 查看简要日志信息
git reflog
1
  1. 查看详细日志信息
git log
1
  1. 查看极简日志信息
git log --oneline
1
  1. 查看最近 n 次的版本信息
git log --all
1
  1. 查看所有分支的版本历史
git log --all
1
  1. 以图形形式展示版本历史
git log --graph
1
  1. 查看涉及到指定文件的commit记录
git log filName
1
  1. 查看指定文件每一行修改对应的commit记录和作者
git blame fileName
1

# 分支命令

# 创建分支

  • 基于当前分支创建分支
git branch 新分支
1
  • 基于指定分支创建分支
git branch 新分支 已有分支
1
  • 基于某个commit创建分支
git branch 新分支  commitID
1
  • 基于当前分支创建分支及并切换到新分支
git checkout -b 新分支
1

# 查看本地分支

git branch -v
1

# 删除分支

  • 安全删除本地分支
git branch -d 分支
1
  • 强制删除本地分支
git branch -D 分支
1

# 切换分支

git checkout 分支名
1

# 合并分支

merge 和 rebase 的区别有待学习实践 😟

  1. 将A分支合并到当前分支，且为 merge 创建 commit
git merge A
1
  1. 将A分支合并到B分支，且为merge创建 commit
git merge A B
1
  1. 把当前分支基于B分支做rebase，以便把B 分支合入到当前分支
git rebase B
1
2
  1. 把 A 分支基于 B分支做rebase,以便把 B　分支合入到A分支
git rebase B A
1

# 版本穿梭与回滚

  1. 工作区指定文件恢复成和暂存区一样
git checkout file1 file2 ...

# tips in the Git Bash
# use "git restore <file>..." to discard changes in working directory
git restore file1 file2 ...
1
2
3
4
5
  1. 暂存区指定文件恢复成和 HEAD 一样
git reset file1 file2 ...

# tips in the Git Bash
# use "git restore --staged <file>..." to unstage
git restore --staged file1 file2 ...
1
2
3
4
5
  1. 工作区和暂存区所有文件恢复成和 HEAD 一样
git reset --hard
1
  1. 工作区和暂存区所有文件恢复成和指定版本一样
git reset --hard 版本号
1

# 修改commit 记录

  1. 往最后一次 commit 追加记录，而不新建 commit - reference (opens new window)
git commit --amend
1
  1. 合并 commit 记录 - reference (opens new window)
git rebase -i HEAD~2
1

# 远程仓库交互

# git remote

  1. 查看所有远程仓库地址别名
git remote -v
1
  1. 为远程仓库起别名
git remote add 别名 地址
1
  1. 删除远程仓库别名
git remote remove 别名
1
  1. 修改别名
git remote rename 旧名 新名
1

# git clone

  1. 克隆远程仓库到本地
git clone 地址
1
  1. 克隆远程仓库指定分支到本地
git clone -b 远程仓库分支名 地址
1

# git pull

  1. 拉取远程分支，并与本地分支合并
git pull 别名 分支名
1

# git push

  1. 推送本地指定分支到仓库指定分支
git push 别名 本地分支:远程分支
1
  1. 如果远程分支被省略，表示将本地分支推送到与之存在追踪关系的远程分支(通常两者同名)，如果该远程分支不存在，则会被新建
git push origin :master
# 等同于
git push origin --delete master
1
2
3
  1. 强制推送
git push --force origin master

或者(强制推送dev)

git push origin dev -f
1

  1. git 命令只提交指定的文件
  1) git add src/xxxxxxxx 添加需要提交的文件名（加路径–参考git status(查看仓库状态)打印出来的路径）
  2) git stash -u -k 忽略其他文件，把现修改的隐藏起来，这样提交的时候就不会提交未被add的文件；
  3) git commit -m ‘fix：xxxxxxxx’；
  4) git pull 拉去合并
  5) git push 推送到远程仓库
  6) git stash pop 恢复之前忽略的文件
